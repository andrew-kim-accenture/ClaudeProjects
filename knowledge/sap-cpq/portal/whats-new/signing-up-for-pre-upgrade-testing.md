Signing Up for Pre-Upgrade Testing

Sign up for Pre-Upgrade testing by creating a new Customer Case through the support channel.

Context

During Pre-upgrade testing, the new release version is deployed to only one Sandbox or Test tenant that you choose. This approach minimizes any impact on ongoing development. You can nominate either the main Sandbox tenant or any additional Sandbox tenants for testing.

Please note: since the number of participants is limited, SAP CPQ reserves the right to shortlist candidates for testing. We also encourage you to submit your requests early.

To sign up for Pre-Upgrade testing, create a new Customer Case through the support channel. In addition to standard details, include the following:

Procedure
Add the <release_identifier> in the subject title, followed by "Pre-Upgrade Testing Request."
Select CEC-SAL-CPQ as the component.
In the case description:
Nominate a tenant for Pre-Upgrade testing.
Provide approval for SAP CPQ to copy the Sandbox tenant data into the Pre-Upgrade environment.
Indicate whether you want the test data created in the Pre-Upgrade testing tenant to be copied back to the Sandbox tenant.
Note

The copying of test data occurs on the first Monday after the Sandbox Upgrade, starting at 09:00 a.m. (CEST/CET). If copying isn't needed, the switchback to the live CPQ system happens on Friday before the Sandbox Upgrade, starting at 4:00 p.m. (CEST/CET).

You can copy the following template to quickly add this information to your Customer Case description:
Example

Approval to copy the Sandbox tenant data into the Pre-Upgrade environment: (Y/N)

Source URL of the Sandbox tenant to be moved to the Pre-Upgrade environment:

Do you want the test data created in the Pre-Upgrade testing tenant to be copied back to the Sandbox tenant? (Y/N)

Results
Once you submit your request, we will review it. If you're shortlisted for Pre-upgrade testing, the onboarding process will start shortly. For one or two weeks, you will have the chance to test the new code. Please report any identified issues as Customer Cases. For seamless resolution and easy tracking, raise all issues as Customer Cases using component CEC-SAL-CPQ with a prefix in the subject line: <release_identifier>.
