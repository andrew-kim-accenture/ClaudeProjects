Quote List Settings

The Quote List Settings administrative section allows SAP CPQ administrators to view and customize the columns displayed to users and set filters for the quick search on the Existing Quotes page in Quote 2.0.

Overview

To access the Quote List Settings section, go to Setup  Quotes  Quote List Settings. Here, you can add, remove, reorder, and modify columns. You can also customize the quote column layout for different user types. In Quote 1.0, the quote list can be managed on Setup  Quotes  Column Headings.

Setting Quote List

Here's the process of setting Quote List:

In the User Type section, add all relevant user types. When you add a new user type, the system adds it to the User Type dropdown menu. The columns and filters for the new user type match the settings for All User Type.

In the User Type dropdown menu, choose the type you want to set the columns layout and filter settings for.

In the Columns Layout section, add columns, choosing among the list of fields in the dropdown menu. Quote Number column is selected by default. Other columns are custom fields and default fields, which include quote information, customer information, and user information.

Note

When you add a column, the system automatically assigns a filter to it.

To define column names, choose the Edit icon for each column.

To set the translation for the column name, choose the Translate this item icon. Choose the relevant dictionary and enter translation value.

Drag the columns into the desired order.

In the Filter Settings section, delete unnecessary ones.

To assign filters for quick search, choose the Edit icon, select the filter operator, and select the checkbox.

Note

You can assign a maximum of three filters for quick search.

Note

By default, Quick Search on the Quote List page displays suggestions for Quote 1.0 only. If you want this functionality for Quote 2.0, contact SAP Support.

Drag the filters into the desired order. To align the filters with the selected columns, choose Reset.

Save your settings, and repeat these steps for other user types.
