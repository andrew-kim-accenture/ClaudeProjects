Password Requirements

Users and administrators can change their own passwords in SAP CPQ (if they aren’t marked as SSO), but administrators can’t change passwords of other SAP CPQ users either via user import/export, user management in the Setup, or API.

Passwords in SAP CPQ  must conform to the following rules:

Must be between 8 and 50 characters long

May contain only the following characters: a-z, A-Z, 0-9

May contain the following symbols: $@!%*#?&

Can't be the same as the previous password

Can't contain your username

These are the default password requirements shipped with every new tenant. If administrators wish to change these requirements on tenant level per their specific business model, they can do so by contacting the SAP CPQ Support team.

Administrators can create temporary password for users in the Setup. After logging in with a temporary password, users are redirected to create a new password.
