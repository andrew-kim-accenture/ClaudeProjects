About Quote 2.0 - Benefits and Compatibility

Quote 2.0 is the new generation quotation engine available since June 2018.

The new quote engine was developed as a part of the initiative to make SAP CPQ the state-of-the-art software that allows users to create large quotes more quickly than in Quote 1.0. After the responsive design was made public in 2015 and the new Document Generation engine made available in 2017, clients needed a new Quote engine that would additionally boost their performance.

Note

Benefits of Quote 2.0:

Performance

Quote 2.0 is way faster than Quote 1.0

Optimized infrastructure for a performant quoting process, including event-based operations, support for "deltas" and a redefined architecture.

Scalability

The most robust support for super large quotes (up to 100k items).

Allows more users to work on the same quote.

Foundation for work on multiple quotes in different tabs.

Future Proof

Obsoleted features are removed from the system and the features that are most frequently in use are redesigned or replaced with powerful alternatives. Additionally, events and pipelines are cleaned up which adds to the improved system performance.

Quote 2.0 is easier to implement than Quote 1.0 due to its streamlined design and associated benefits.

Solid foundation for future quote improvements.

Sustainable

Quote 2.0 is API driven and easier for establishing integration with external tools.

The stateless approach ensures scalability when processing quotes.

Enhanced flexibility through scripts and webhook events enables organizations to adapt the system to meet their business requirements.

A refined and sustainable architecture ensures the quoting engine remains efficient over time.

The following integrations are supported in Quote 2.0 engine:

SAP S/4HANA Cloud

External Configurator Integration

SAP Subscription Billing

SAP Billing Revenue and Innovation Management

SAP Master Data Service for Business Partners

Compatibility

Before initiating the migration from Quote 1.0 to Quote 2.0, check the following list to understand the compatibility of the new quote engine with the rest of the system:

Quote 1.0 and Quote 2.0 can’t be used in parallel on the same tenant.

Only responsive design is supported in Quote 2.0.

The user interface is the same, although some tweaks have been made (for example, Load Existing Project/Quotation is redesigned).

Quote 2.0 shouldn’t be enabled on existing tenants if the system hasn’t been previously adjusted to Quote 2.0. This requires a migration project.

Some features aren’t available in Quote 2.0. They’re either obsoleted or they’re replaced with an improved alternative. This is described in Quote 1.0 vs Quote 2.0 - Feature Comparative Overview.

Old Quotes can be migrated to the new engine.

There is no out-of-the box solution for complete migration (including migration of configuration), migration must be done in a project model. Moving from Quote 1.0 to Quote 2.0 should be handled in a project.
