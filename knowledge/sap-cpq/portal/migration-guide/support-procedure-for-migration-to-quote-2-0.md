Support Procedure for Migration to Quote 2.0

This document outlines the support procedure to be followed during the migration to Quote 2.0. It provides guidance for each step in the, Quote 2.0 Migration Process where interactions with SAP Support is required.

1. Initiate Migration

To start the migration, please create a ticket with SAP support indicating your intention to begin the process. This ticket will serve as the central hub for communication and coordination of the next steps

2. Implementation Process
Copy Production Quote 1.0 Tenant

Initiate the copying of the Quote 1.0 production tenant through the migration case created earlier

Input needed: Details of the source tenant (Quote 1.0 production tenant) and the target tenant (dedicated for Quote 2.0 implementation)

Activate Quote 2.0

Activate Quote 2.0 on the tenant dedicated for its implementation through the migration case.

Input needed: Tenant details

Activate Migration Tool

Activate the Migration tool on the tenant dedicated for Quote 2.0 implementation through the migration case.

Input needed: Tenant details

3. Switching to Production with Quote 2.0
Copy Production Quote 1.0 Tenant

Initiate the copying of the Quote 1.0 production tenant through the migration case.

Input needed: Details of the source tenant (Quote 1.0 production tenant) and the target tenant (for final preparation and validation before switching to production).

Activate Quote 2.0

Activate Quote 2.0 on the tenant dedicated for final preparation and validation through the migration case.

Input required: Tenant details

Activate Migration Tool

Activate the Migration tool on the tenant dedicated for final preparation and validation through the migration case.

Input required: Tenant details

Cut-off Transition

Preparation for Cut-off

Initiate preparation activities for the cut-off transition to Quote 2.0 through the migration case.

Input needed: Details about the Quote 2.0 tenant that will become productive and official approval to copy this tenant's database to the productive environment.

Rollback

To ensure no risk to your business processes, indicate that a backup of the Quote 1.0 productive tenant should be done through the migration case.

Input needed: Details about the Quote 1.0 productive tenant.
