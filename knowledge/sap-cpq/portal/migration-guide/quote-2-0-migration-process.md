Quote 2.0 Migration Process

Following document describes recommended migration path for clients who decided to migrate their Quote 1.0 model to the Quote 2.0 engine and need guidance on the required steps. Migration to the Quote 2.0 engine needs to be handled as a project, and the goal of this document is to provide needed information about each step in order to ensure smooth transition to new engine.
Note

At the start of your migration journey, it's important to define the goal you want to achieve with the migration, as this significantly impacts your migration path. There are two basic scenarios:

Switching to Quote 2.0 without significant change in their modeling

The only model adjustment needed is related to adopting Quote 2.0 vs Quote 1.0 differences .

This approach is recommended if you want to switch to Quote 2.0 while keeping your current model. If necessary, you can change the model after completing the migration. With this approach, you can use the migration tool and have your old Quotes available after migration.

You can find more details in the Migration Guide

Switching to Quote 2.0 with a model rebuild:

This approach is recommended if you want to rebuild and optimize your model when switching to Quote 2.0.

However, you cannot use the migration tool or have your old Quotes available after migration, as the prerequisites for using the migration tool cannot be ensured with a model rebuild. This document does not cover this scenario.

If you're considering this approach, please reach out to SAP for assistance in defining the commercial and implementation aspects of the migration.

Note

The sequence of recommended steps is not mandatory and can be adjusted to suit your specific needs . Steps marked as mandatory must be fulfilled as they are prerequisite for successful usage of migration tool.

Note

In order to ensure that existing quotes can be used and referenced once migration to new engine is completed, SAP developed Quote Migration Tool should be used for copying quotes from Quote 1.0 to Quote 2.0.
