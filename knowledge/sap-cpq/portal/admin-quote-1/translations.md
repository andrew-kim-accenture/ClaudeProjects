Translations

There are certain entities in SAP CPQ that you can create on your own, such as individual products and custom fields. These entities can be translated so that administrators and users from other countries can understand what they are and how they can be used.

Products, Attributes, Attribute Values, Categories, Product Types, Guided Selling and Product Tabs in Product Catalog.

Shipping and Promotions/Special Pricing in Pricing/Calculations.

Document Generation Templates, Custom Fields, Search Fields, Custom Quote Tabs and Column Headings in Quotes.

User Custom Fields in Users

Statuses, Actions, Item Actions, Approval Branch and Approval Rules in Workflow/Approval

Customer Custom Fields in Customers/Customer Roles

Countries/Regions in General

You can translate an item by clicking the Translate this item action next to it, then select the language, or languages, you wish to translate it to.

Note

Multilanguage translations can't be added to SAP CPQ via API. You need to add them manually.
