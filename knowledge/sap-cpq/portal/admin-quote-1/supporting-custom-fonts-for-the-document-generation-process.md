Supporting Custom Fonts for the Document Generation Process

Follow these instructions to upload and utilize fonts other than those available in your Microsoft Word templates. This will ensure that all fonts used in template documents are also applied to the generated output document.

Customers have the option to utilize any of the standard system fonts in their Microsoft Word templates in the .doc and .docx formats prior to processing. Any fonts selected from Microsoft Word during pre-processing will be consistently used and applied when generating output documents in the Word format.

When generating documents in PDF format, Aspose may not recognize the chosen custom fonts and the fonts in output file can be replaced with some of default fonts (Arial, Times New Roman, and so on).

Custom fonts can be uploaded through file management  Setup/File Management/Fonts.

If some custom fonts are missing, a warning message displays in the event log.

SAP CPQ supports only True Type Fonts (.ttf) and Open Type Fonts (.otf).
